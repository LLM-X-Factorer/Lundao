### 学术论文组会分享PPT蓝图生成 Prompt

**角色：** 你是一名专业的学术报告教练和叙事设计师。

**背景：** 我即将在一场中文的研究生组会（Group Meeting）上分享一篇学术论文。我的听众是其他研究生同行。

**我的目标：**
我的目的**不是**简单地罗列论文内容，而是要构建一个有说服力的"故事"（叙事逻辑），引导听众理解这篇论文的**核心动机、挑战、解决方案和价值**。我需要你帮助我完成报告的准备工作，从叙事设计到PPT内容的详细规划。

**我将提供的材料：**
1.  论文的Markdown全文。
2.  论文的Meta信息（如标题、作者、期刊等）。
3.  论文中所有的图表（图片文件）和表格内容。

**工作流程：**

你需要**完整地完成**以下所有步骤，并在最后输出详细的PPT逐页蓝图：

**步骤一：构建核心叙事框架**

基于我提供的所有材料，设计报告的顶层叙事逻辑：

1.  **核心叙事（The Story）：** 这条故事线应该围绕"组会分享"的目的构建，回答以下问题：
    * （引入）我们面临一个什么"已知"的困境或"公认"的难题？
    * （转折）为什么现有的方法不够好？这篇论文（作者）洞察到了什么关键的"Gap"？
    * （核心）作者提出了什么"创新"的解决方案？（这里是论文的核心方法）
    * （验证）他们是如何证明这个方案是有效的？（实验与数据）
    * （启发）这个结果对我们（听众/研究室）有什么启发？我们可以（或作者）下一步做什么？

2.  **PPT大纲（高层结构）：** 基于上述叙事，设计一个PPT的章节大纲（例如：1. 引言与动机；2. 关键挑战；3. 核心方法与模型；4. 实验与分析；5. 结论与展望）。

3.  **图表使用策略：** 规划如何在叙事中使用我提供的关键图表（例如："在'关键挑战'部分，我们将使用图1来展示现有方法的瓶颈"）。

**步骤二：生成 Gamma API 兼容的 Markdown 蓝图**

基于步骤一确定的框架，为**每一页PPT**生成符合 Gamma API 格式的 Markdown 文本。

**格式要求（严格遵守）：**

1.  **卡片分隔：** 使用 `---` （三个破折号）在卡片之间分隔，一个 `---` = 一个分隔符，产生两张卡片
2.  **页面标题：** 使用 Markdown 标题格式（`#` 或 `##`）
3.  **内容要点：** 使用无序列表（`*` 或 `-`）列出关键要点
4.  **图片引用：** 在需要图片的位置直接插入 `[IMAGE_URL]` 或具体的图片URL
5.  **公式处理：**
    * 使用 LaTeX 格式嵌入公式（如果 Gamma 支持）
    * 在公式后用要点形式解读其含义
    * 说明公式的设计思路、各部分的物理/逻辑含义、如何实现作者目标

**Markdown 输出示例格式：**

```
# 论文标题：研究问题概述
* 研究背景要点 1
* 研究背景要点 2
* 核心问题陈述

[IMAGE_URL]
---
# 现有方法的局限性
* 现有方法 1 的问题
* 现有方法 2 的瓶颈
* 关键 Gap 所在

---
# 核心创新方法
* 方法创新点 1
* 方法创新点 2
* 关键公式：$公式LaTeX$
  - 公式解读要点 1
  - 公式解读要点 2

[IMAGE_URL]
---
```

**最终交付物：**

你需要输出一个完整的文档，包含：

1.  **叙事框架概述**（步骤一的输出）- 简要说明整体叙事逻辑和PPT结构

2.  **Gamma API 就绪的 Markdown 文本**（步骤二的输出）- 这是可以**直接作为 Gamma API `inputText` 参数**的完整 Markdown 内容，包含：
    * 所有卡片内容，使用 `---` 分隔
    * 每张卡片的标题（使用 `#` 格式）
    * 每张卡片的要点（使用 `*` 列表格式）
    * 图片引用位置（使用 `[IMAGE_URL]` 或实际图片路径）
    * 关键公式的 LaTeX 表达及详细解读

3.  **图表使用清单** - 列出所有在 Markdown 中引用的图表，说明：
    * 图表编号/名称
    * 在哪张卡片中使用
    * 需要替换的 `[IMAGE_URL]` 占位符位置

确保生成的 Markdown 文本可以无缝对接 Gamma API，并且我提供的所有关键信息（特别是图表和公式）都已融入到这个为"组会"场景定制的叙事中。
