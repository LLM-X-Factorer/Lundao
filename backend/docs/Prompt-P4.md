# 演讲稿“负面清单”（严格禁止项）
为了杜绝“机器味”，请严格遵守以下禁令：
1.  **禁止“报幕式”过渡：** 严禁说“接下来让我们看看下一页”、“这张幻灯片展示了...”、“如图所示”。（人类会说：“那为什么会出现这个问题呢？”或者直接切入话题）。
2.  **禁止书面语硬读：** 严禁出现“我们要进行彻底的分析”、“该方法具有显著的鲁棒性”这种书面长句。请换成口语：“我们要好好挖一下这个问题”、“这个方法特别抗造”。
3.  **禁止空洞的赞美：** 严禁使用“赋能”、“颠覆性”、“完美的”、“令人震惊的”。用数据和事实说话。
4.  **禁止僵硬的列表：** 不要说“第一点...第二点...第三点...”。人类会说：“首先...紧接着...最后...”。

# 演讲逻辑与口语技巧要求

**1. 强互动感的开场与过渡 (Hooks & Bridges)**
* 每一页的开头，不要直接念标题。要用**设问**或**承接上一页的逻辑**来开启。
* *示例：* 不要说“现在讲方法论。” 要说：“既然传统的方法走不通，那我们是怎么做的呢？大家请看...”

**2. 视觉引导 (Visual Anchoring)**
* 当蓝图提到图表时，你的脚本必须**指挥观众的眼睛**。
* *指令词：* “大家请看左上角的数据...”、“注意这条红色的曲线...”、“随着横坐标时间的推移...”
* *目的：* 让听众觉得你在指着屏幕讲，而不是在背稿子。

**3. 公式即直觉 (Math as Intuition)**
* **绝对不要**把公式里的希腊字母一个个读出来（除非为了指代）。
* **解释直觉：** 告诉听众这个公式**“想干什么”**。
* *示例：* 不要说“P等于阿尔法乘以...” 要说：“这个公式的核心其实就是一个过滤器，它把不相关的噪音全部筛掉，只留下...”

**4. 短句与停顿 (Rhythm)**
* 多用短句。人类说话是一截一截的。
* 关键观点后，适当留白（可以在稿子里标注 `(停顿)` ）。
