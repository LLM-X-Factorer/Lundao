# Feature #005: PPT内容智能化 - 调研报告

**调研日期**: 2025-10-16
**调研目标**: 理解学术PPT生成的最佳实践，为论导Lite设计真正解决科研问题的PPT内容策略
**调研方法**: 互联网搜索 + 竞品分析 + 最佳实践研究

---

## 一、问题背景

### 当前系统状态

**技术实现** ✅ 已完成：
- LaTeX公式渲染（KaTeX）
- 代码语法高亮（7种语言）
- Markdown渲染管道
- 水印系统
- 固定高度幻灯片容器（600px）
- 键盘导航

**内容质量** ❌ 存在不足：
- Mock数据仅为演示用途，未针对实际科研场景优化
- 缺乏"阅读论文→理解论文→创建PPT→组会分享"的完整叙事结构
- 无法体现论导Lite真正解决科研过程问题的能力
- PPT内容未遵循学术汇报的最佳实践

### 用户场景

**目标用户**: 硕士/博士研究生、青年科研工作者

**典型工作流**:
1. 发现感兴趣的论文（arXiv/顶会）
2. 阅读并理解论文核心内容
3. 准备组会汇报（15-20分钟）
4. 讲解论文给导师和同学

**核心痛点**:
- 从论文到PPT的转换耗时（平均2-4小时）
- 不知道如何提取核心创新点
- PPT结构混乱，缺乏叙事逻辑
- 内容密度控制不佳（要么太详细，要么太简略）

## 三、学术PPT最佳实践

### 3.1 内容结构规范

#### 标准结构（10-15页）

```
1. 封面页 (1页)
   - 论文标题
   - 作者/会议/年份
   - 汇报人信息

2. 目录/大纲 (1页)
   - 研究背景
   - 核心方法
   - 实验结果
   - 总结展望

3. 研究背景 (2-3页)
   - 问题是什么？
   - 为什么重要？
   - 现有方法的局限

4. 核心创新点 (1页) 🔥 关键
   - 本文提出了什么？
   - 与已有方法的本质区别
   - 3-5个bullet points

5. 方法详解 (2-3页)
   - 核心技术/算法
   - 系统架构图（如适用）
   - 关键公式（简化）

6. 实验结果 (2-3页)
   - 数据集说明
   - 关键性能指标（表格/图表）
   - 消融实验（如有）

7. 案例展示 (1页, 可选)
   - 可视化效果
   - 实际应用示例

8. 总结与启发 (1页)
   - 主要贡献
   - 局限性
   - 未来方向

9. 致谢/Q&A (1页)
```

**总计**: 10-13页（符合15分钟汇报标准）

### 3.2 内容密度原则

#### 5×5规则
- 每句话不超过5个单词（中文：10个字）
- 每页不超过5行文字

#### 1分钟原则
- 1页幻灯片 = 1分钟讲解
- 15分钟汇报 = 最多15页（建议10-13页，留2分钟讨论）

#### 视觉优先原则
- **禁止**: 直接粘贴论文表格、大段文字、文献综述
- **推荐**: 图表、公式简化版、bullet points、架构图

#### 字号规范
- 标题：≥36pt
- 正文：≥28pt
- 图表标注：≥24pt

### 3.3 叙事框架（科研Storytelling）

#### 经典结构：问题→方法→结果→意义

```
[幕起] 背景 + 问题陈述
   ↓
[矛盾] 现有方法的局限性
   ↓
[转折] 本文的核心创新（3-5个关键点）
   ↓
[行动] 技术方法详解（简化版）
   ↓
[高潮] 实验结果（对比表格/图表）
   ↓
[解决] 贡献总结 + 未来展望
```

#### 6个情节元素（改编自经典叙事理论）

1. **角色**: 研究问题（拟人化）
2. **背景**: 研究领域现状
3. **张力**: 现有方法vs.理想目标的差距
4. **行动**: 论文提出的方法
5. **高潮**: 实验验证（性能提升）
6. **解决**: 对领域的贡献

**效果**: 研究表明，叙事化的学术PPT可提升**30%**的听众留存率

### 3.4 组会汇报特殊要求

#### 与会议报告的区别

| 维度 | 会议报告 | 组会汇报 |
|------|---------|---------|
| 时长 | 20-30分钟 | 10-20分钟 |
| 听众 | 同行专家 | 导师+组员（背景差异大） |
| 深度 | 技术细节 | 核心思想为主 |
| 互动 | Q&A集中在最后 | 随时可被打断提问 |
| 目的 | 展示成果 | 学习+讨论启发 |

#### 组会PPT的独特要求

1. **背景讲解更充分**（2-3页 vs. 会议的1页）
   - 组员可能不熟悉该子领域
   - 需要解释术语和基础概念

2. **创新点可视化**
   - 用对比图、before/after示意图
   - 而非纯文字列举

3. **启发性讨论**
   - 最后1-2页：对我们的研究有何启发？
   - 可以借鉴的方法？
   - 潜在的改进方向？

4. **简洁但不简陋**
   - 避免花哨动画
   - 专业学术风格（非答辩模板风格）

## 四、AI内容生成策略分析

### 4.1 当前AI工具的内容生成方式

#### PPTAgent（中科院）的两阶段方法

```
Stage 1: 大纲生成
输入: 论文PDF/文本
   ↓
LLM提取: 标题、关键段落、图表
   ↓
输出: 结构化大纲（JSON）

Stage 2: 幻灯片生成
输入: 大纲 + 论文内容
   ↓
LLM生成: 每页的标题、bullet points、图表引用
   ↓
输出: Markdown格式幻灯片
```

**优点**: 结构清晰、质量可控
**缺点**: 需要完整PDF上传

#### 笔灵AI的快速生成方式

```
输入: 论文摘要/全文
   ↓
模板匹配: 识别论文类型（实验型/理论型/综述型）
   ↓
内容填充: 将论文段落映射到模板slots
   ↓
输出: 完整PPT（20-30页）
```

**优点**: 速度快（2分钟）
**缺点**: 模板化严重、内容冗余

### 4.2 核心创新点提取算法

#### 基于LLM的提取方法

```python
# 伪代码示例
prompt = """
分析以下论文的核心创新点，提取3-5个关键贡献。

论文标题: {title}
摘要: {abstract}
引言: {introduction}
方法: {method}

要求：
1. 每个创新点用一句话概括（不超过20个中文字）
2. 按重要性排序
3. 突出与已有方法的本质区别
4. 使用学术术语但避免过度专业化

输出格式（JSON）:
{
  "innovation_points": [
    {"title": "...", "description": "...", "importance": "high"},
    ...
  ]
}
"""
```

#### 论文结构解析规则

常见创新点位置：
1. **Abstract**: "We propose...", "Our method...", "Contributions:"
2. **Introduction最后一段**: "Our main contributions are..."
3. **Method章节标题**: 新算法/框架的名称
4. **Results**: 性能提升的量化指标

### 4.3 内容密度控制

#### 自适应页数生成

```
IF 会议类型 == "组会汇报" AND 时长 == 15分钟:
    目标页数 = 10-13页
    背景页 = 2-3页
    方法页 = 2-3页
    结果页 = 2-3页
    其他 = 3-4页

ELSE IF 会议类型 == "学术会议" AND 时长 == 20分钟:
    目标页数 = 15-18页
    ...
```

#### 内容精简策略

| 原始内容类型 | 精简策略 | 示例 |
|-------------|---------|------|
| 大段文字 | 提取关键句 → bullet points | "本文提出..." → "✓ 提出XXX方法" |
| 复杂表格 | 保留关键列/行 | 10列 → 3-4列（baseline + 本文 + SOTA） |
| 数学公式 | 简化为核心公式 | 完整推导 → 最终形式 |
| 文献综述 | 删除 | - |
| 消融实验 | 仅保留关键对比 | 8组实验 → 2-3组最重要的 |

## 六、关键发现总结

### 6.1 内容生成的核心要素

1. **结构化大纲** 🎯
   - 10-13页标准结构
   - 每页1分钟讲解时长
   - 符合学术汇报规范

2. **创新点提取** 🎯
   - 从Abstract/Introduction/Method自动识别
   - 3-5个bullet points
   - 突出与已有方法的区别

3. **内容精简** 🎯
   - 5×5规则（5词/句，5行/页）
   - 删除文献综述、复杂表格
   - 保留核心公式（简化版）

4. **叙事连贯** 🎯
   - 问题陈述 → 方法创新 → 结果验证 → 贡献总结
   - 每页之间有逻辑衔接

5. **视觉优化** 🎯
   - 图表优先于文字
   - 字号≥28pt
   - 专业学术风格

### 6.2 竞品差异化机会

论导Lite的独特优势：

1. **已有论文分析数据** ✨
   - 中文摘要（AI生成）
   - 创新点（已提取3个）
   - 论文元数据（标题、作者、领域）
   - **无需用户重新上传PDF或输入信息**

2. **场景专注** ✨
   - 专门为"15分钟组会汇报"优化
   - 而非通用答辩/会议报告

3. **零摩擦** ✨
   - 直接从arXiv论文"一键生成PPT"
   - 无需注册、无需配置

4. **中文优化** ✨
   - 学术术语中文翻译精准
   - 符合国内研究生汇报习惯

### 6.3 技术可行性评估

| 功能模块 | 复杂度 | 数据依赖 | 优先级 |
|---------|--------|---------|--------|
| PPT结构生成器 | 中 | 论文元数据 | P0 |
| 创新点提取（利用现有数据） | 低 | AI分析结果（已有） | P0 |
| 内容精简算法 | 中 | 中文摘要（已有） | P1 |
| 叙事线生成 | 高 | 需要调用LLM | P1 |
| 图表位置建议 | 高 | 需要解析论文PDF | P2 |

**MVP范围建议**: P0 + 部分P1（约2-3周开发）
